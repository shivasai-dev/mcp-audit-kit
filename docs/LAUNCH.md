# Launch Plan

## Target users

- MCP server maintainers.
- Developers adding tools to local AI coding assistants.
- Security-minded teams experimenting with agent workflows.

## Release checklist

- Keep the README focused on the concrete problem.
- Add one screenshot or short GIF of the UI after the first GitHub push.
- Tag the first release as `v0.1.0`.
- Publish a short post with a risky manifest before/after.
- Ask MCP builders for rule ideas, not stars.

## Communities

- GitHub topic pages for `mcp`, `model-context-protocol`, `ai-agents`, `security`, and `developer-tools`.
- Hacker News "Show HN" after the CLI and SARIF output are stable.
- LinkedIn post framed around "reviewing agent tools before they run."
- MCP Discord/community channels if self-promotion rules allow it.

## Star strategy

Stars should come from usefulness, not tricks:

- Ship a working demo.
- Make the README immediately clear.
- Add fixtures people can recognize.
- Keep issues labeled and easy to pick up.
- Publish a roadmap with small contributor-friendly tasks.
